Beschleunigung von Python-Implementationen

Ein Projekt, über das ich auch meine Bachelorarbeit geschrieben habe, beschäftigt sich mit den Beschaffenheiten Python, besonders, wie eine Python Implementation ausgeführt wird und wie dies unter Umständen zu beschleunigen ist. Primäres Testobjekt des Projektes war dabei eine Implementation des relativ bekannten Conways Game of Life.

Hierbei war der Fokus nicht die algorithmische Geschwindigkeit im Sinne der O() Notation, sondern das Ausführen mittels anderer Python Implementationen als der standardmäßig verwendeten CPython Implementation. Hierbei spielt zum einen die Auslagerung von Teilen des Codes nach C mittels Cython eine Rolle. Außerdem die Auslagerung nach C++ und dem entsprechenden g++ Compiler auf klassische Art und Weise mit pybinds11.

Als zweiten Schritt wurde dann numba als Möglichkeit der "Auslagerung" in eine andere Ausführungsart innerhalb von Python betrachtet. Hierbei wird spezifisch der Python Interpreter umgangen, indem Just-In-Time mit der LLVM Architektur kompiliert wird. Betrachtet wurde hier neben der Geschwindigkeit auch Kompatibilität als wichtiger Faktor.

Als letztes wurde dann die Auslagerung der Code Ausführung auf die GPU betrachtet. Dies stellt für Python eine wirklich hervorragende Möglichkeit dar. So wird spezifisch herausgestellt, wie (verhältnismäßig) leicht sich aus Python auf der GPU programmieren lässt. Gleichzeitig wird auch betrachtet, dass die Installation und das Setup, gerade unter Windows, einen durchaus beträchtlichen Aufwand darstellen kann. Nachdem diese aber einmal durchgeführt wurde, in Fall des Projektes mit Nvidias CUDA, gibt es für Python Bibliotheken, die die Arbeit aus Python heraus enorm komfortabel gestalten. Spezifisch wurde im Rahmen des Projektes herausgestellt, wie viel schneller eine GPU Ausführung ist, auch im Vergleich mit einer C++ Ausführung.

Spotify-Applikation mit React

In dem Bestreben, React weiter zu erkunden und zu lernen, habe ich ein Projekt entwickelt, dass React für eine Webapplikation benutzt, aus der aus auf die Spotify API zugegriffen werden kann. Die grundlegende Funktionalität der App ist dabei recht simpel, so ermöglicht diese das Zusammenstellen einer Playlist. Hierfür kann zunächst ein gewünschter Titel gesucht werden. Anschließend werden für diesen Titel entsprechende Spotify Ergebnisse vorgestellt. Titel aus den Ergebnis können dann wiederum zu einer Playlist hinzugefügt werden. Diese kann umbenannt werden und ist anschließend verfügbar.

Vorraussetzung für die Nutzung der App ist ein funktionierender Spotify Account. Die Anmeldung erfolgt über ein OAuth2 Verfahren, bei dem man sich bei Spotify selbst anmelden muss. (Geschieht automatisch beim ersten Suchdurchlauf nach einem Titel). Das OAuth2 Verfahren erlaubt dabei eine vollständig Front-End basierte Nutzung der API ohne die Gefahr, API Keys zu gefährden. Erstellte Playlists stehen anschließend unter dem Account zur Verfügung.

Die App ist ansonsten Work-in-Progress. Weitere Funktionalitäten, sowie eine etwas verbesserte User-Experience beim Anmeldeprozess sind aktuell in Arbeit.
